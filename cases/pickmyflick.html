Work / PickMyFlick

Case 01 · Senior Design Project

A recommendation system I could evaluate, not just demo.

PickMyFlick combines several recommendation signals, stores real user activity and explains why a title was suggested.

Python · Flask · React · PostgreSQL · scikit-learn · Sentence Transformers

My role

Full-stack developer and project owner

Outcome

Working hybrid recommender with offline evaluation

Repository

Private — evidence is summarized in this case study

The problem

A movie list based only on popularity is easy to build, but it does not become personal when a user rates films. I wanted recommendations that used several kinds of evidence and could still explain why each title appeared.

What I built

I built a Flask API and React interface around a 10,000-film catalogue, with PostgreSQL storing accounts, ratings and recommendation events. The application includes searchable films, rating history, similar-film exploration and short recommendation reasons.

The ranking combines collaborative filtering, TF-IDF content similarity, semantic movie embeddings and a confidence score.

The main decision

I used a balanced hybrid rather than trusting one model. The live score weights collaborative evidence at 40%, TF-IDF at 25%, embeddings at 25% and confidence at 10%.

Embeddings are precomputed so normal use does not depend on an external model call. If those files are unavailable, the system falls back to the TF-IDF calculation instead of failing completely.

What came of it

The result is an end-to-end recommender with authentication, saved ratings and explanations. I evaluated the approaches offline on 603 MovieLens users. At ten recommendations, the balanced hybrid reached a 47.93% hit rate and 0.1079 NDCG, outperforming the individual content, embedding and collaborative baselines on those measures.

Evidence10,000-film catalogue · 603 users evaluated · content, embedding, collaborative and hybrid models compared · saved metrics and chart report

What I would improve

The next portfolio pass needs clearer screenshots of rating, recommendation and explanation flows. I would also make the evaluation protocol easier to inspect directly from the case page.
