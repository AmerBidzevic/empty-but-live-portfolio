A habit loop connected to real application state.

GrowLoop turns routines, completions and progress into a working web application rather than a set of disconnected interface mockups.

The problem

A habit tracker needs more than a checkbox. Users need a clear daily loop, a record of what they completed and enough feedback to understand progress without turning the interface into a game they cannot control.

What the team built

GrowLoop supports registration and login, onboarding, habit CRUD, pause and resume, completion history, XP, streaks, levels, achievements, analytics, notification preferences and recommendation logic.

The frontend talks to a Python REST API backed by SQLite. Server-side session tokens protect logged-in calls, and passwords are stored as PBKDF2 hashes.

The structural decisions

The project separates API routes, services, repository access and persistent storage. Database queries stay inside the repository instead of spreading through handlers. Recommendation behavior is split into strategies for habit suggestions, burnout detection, adaptive difficulty and workload guidance.

What came of it

The release connects the entry flow shown above to actual habit, progress and analytics behavior. It includes five automated backend/API tests and deployment configuration with a persistent SQLite disk path.

EvidenceAuthentication · habit CRUD · completion history · XP and streaks · analytics · recommendation strategies · 5 automated API tests

What I would improve

The case needs clearer ownership notes for individual team contributions and screenshots beyond the entry flow. The recommendation rules also need user testing before being described as genuinely helpful rather than simply implemented.
